# Chrome 插件部署指南

本项目已配置为支持生成 Chrome 扩展程序，并作为**新标签页**运行。

## 1. 生成扩展程序文件

您刚才已经生成了 `manifest.json`。现在我们需要重新构建项目，以确保 `manifest.json` 被包含在最终的输出目录中。

在终端中运行：

```bash
npm run build
```

构建完成后，`out` 文件夹就是完整的 Chrome 插件包。

## 2. 在 Chrome 中加载插件

1.  打开 Chrome 浏览器，在地址栏输入：`chrome://extensions/` 并回车。
2.  在右上角开启 **"开发者模式" (Developer mode)** 开关。
3.  点击左上角的 **"加载已解压的扩展程序" (Load unpacked)** 按钮。
4.  在弹出的文件选择框中，选择本项目根目录下的 **`out`** 文件夹。
5.  点击 "选择文件夹"。

## 3. 验证

1.  安装成功后，打开一个新的标签页 (Cmd+T / Ctrl+T)。
2.  Chrome 可能会提示 "此扩展程序更改了您的新标签页"，点击 "保持现状" 或 "Keep it" 确认即可。
3.  您现在应该能看到您的导航页了！

## 注意事项

*   **样式/脚本问题**：由于 Chrome 插件 (Manifest V3) 对内联脚本 (Inline Scripts) 有严格的安全限制，Next.js 生成的一些用于页面交互的脚本可能会被拦截。
    *   **现象**：控制台可能会有红色 CSP 报错。
    *   **影响**：如果是纯静态导航（只包含链接跳转），通常**不会影响使用**，因为 HTML 和 CSS 是能正常加载的。点击链接依然有效。
*   **更新代码**：如果您修改了 `app/page.tsx` 或其他代码，需要重新运行 `npm run build`，然后在 `chrome://extensions/` 页面点击该插件卡片上的 **刷新图标** 才能生效。
